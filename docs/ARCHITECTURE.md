# 🏛️ Architecture Overview

CodeHarness bridges the gap between **autonomous AI coding** and **human oversight**. It wraps raw agent execution within a controlled, local execution environment managed via a streamlined Streamlit user interface.

## System Topology

CodeHarness is composed of four primary layers:

1. **The Presentation Layer (UI)**: A Streamlit frontend providing live chat, configuration, and approval dialogs.
2. **The Orchestration Layer**: A Python package (`github_harness.py`) that handles GitHub cloning, branch creation, and triggering the Swarm.
3. **The Cognitive Layer (Agents)**: A Swarm of CrewAI agents (Explorer, Coder, Tester) communicating over shared LLMs.
4. **The Execution Sandbox**: An abstraction for running untrusted shell commands and tests (either locally or via E2B Cloud VMs).

```mermaid
graph TD
    UI[🖥️ Streamlit UI] --> |GitHub URL + Params| ORCH[⚙️ Orchestrator (github_harness.py)]
    ORCH --> |Prepare Workspace| GIT[🐙 GitHub API & Local Git]
    GIT --> |Clone Repo| WS[📁 Dynamic Workspace]
    ORCH --> |Initialize Process| CREW[🤖 CrewAI Swarm]
    
    subgraph Cognitive Layer
        CREW --> EXP[🔍 Explorer Agent]
        CREW --> COD[💻 Coder Agent]
        CREW --> TST[🧪 Tester Agent]
    end
    
    EXP --> |Read| WS
    COD --> |Read/Write| WS
    TST --> |Execute Tests| SANDBOX[🛡️ Sandbox (Local/E2B)]
    SANDBOX --> |Stdout/Stderr| TST
    
    COD -.-> |Request Write Permission| HOOK{Gated Approval Hook}
    HOOK -.-> |Wait for User| UI
    UI -.-> |Approve/Deny| HOOK
```

> [!NOTE]
> **Why CrewAI?** We utilize CrewAI because it provides a modular, hierarchical architecture for defining specialized agents with specific tool access, rather than a monolithic all-knowing loop.

## The Presentation Layer (Streamlit)

Located in `src/codeharness/ui/app.py`, the frontend handles:
- **Thread-Safe Callbacks**: Streamlit executes sequentially from top to bottom. Agent logic runs in a separate background thread, communicating with the UI via global state lists and threading events.
- **Credential Management**: Environment variables and API keys are injected dynamically at runtime to prevent them from leaking into the git history.

## The Dual-Execution Sandbox

Untrusted code generated by AI can be dangerous. CodeHarness implements a "Dual-Execution Sandbox" abstraction in the testing layer.

> [!WARNING]
> By default, if `USE_LOCAL_TESTING=true` is set, code is executed directly on your host machine via Python's `subprocess`. Ensure you trust the codebase you are modifying.

If E2B is configured (`E2B_API_KEY` provided), CodeHarness automatically routes all `run_tests` and execution commands to an ephemeral, cloud-based micro-VM using `crewai_tools.E2BExecTool`. This completely isolates terminal damage and provides a secure playground for the agent.
