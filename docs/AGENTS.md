# 🤖 The Cognitive Layer: Swarm Configuration

CodeHarness operates using a multi-agent "Swarm" managed by CrewAI. Instead of having a single prompt try to write code, read files, and test things (which leads to hallucinations), CodeHarness delegates these tasks to highly constrained specialists.

## The Agent Roster

CodeHarness currently employs three specialized agents defined in `src/codeharness/core/code_harness.py`.

### 1. 🔍 The Explorer
- **Role**: Codebase Explorer
- **Goal**: Map the repository and surface relevant files for the task.
- **Permissions**: **Read-Only** (`read_file`, `list_dir`).
- **Philosophy**: This agent builds context. It is strictly forbidden from writing code. By forcing the system to read the disk first, we eliminate the AI's tendency to guess file structures.

### 2. 💻 The Coder
- **Role**: Staff-Level Software Engineer
- **Goal**: Implement the requested change by editing files on disk.
- **Permissions**: **Read & Write** (`read_file`, `write_file`, `list_dir`).
- **Philosophy**: This agent executes changes. It operates with `reasoning=True` enabled in CrewAI, meaning it executes a chain-of-thought planning phase *before* attempting file modifications. It is constrained by negative prompts (e.g., "DO NOT assume file state").

### 3. 🧪 The Tester
- **Role**: Test Runner
- **Goal**: Run the project's tests in the sandbox and report pass/fail with output.
- **Permissions**: **Execution** (Sandbox execution tools, `read_file`).
- **Philosophy**: This agent operates the terminal. It runs `pytest` and returns the actual `stdout/stderr` traces to the manager. It acts as an objective reality check against the Coder's work.

---

## 🧠 The Manager (Hierarchical Orchestrator)

When a GitHub Issue is passed into CodeHarness, it is handed to a CrewAI **Process.hierarchical** manager. 

The Manager Agent is automatically generated by CrewAI using your chosen LLM (e.g., GPT-4, Claude 3.5 Sonnet, etc.). The Manager evaluates the task, delegates sub-tasks to the Explorer to gather context, assigns the Coder to make the edits, and delegates verification to the Tester. 

This loop repeats until the Manager is satisfied that the GitHub issue is completely resolved and tested.

> [!TIP]
> **Tuning the Persona:** The system prompts and personas are the "psychometric tuning" of your agents. Changing the Coder from "Software Engineer" to "Staff-Level Software Engineer" was found to mathematically reduce careless errors during file edits.
